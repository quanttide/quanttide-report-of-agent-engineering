# Write Report — 量潮技术报告写作技能

## 四条原则

### 一、自己买单

写判断要带依据、带数据、带事实。不接受空泛的「我觉得」。

- 批评一个模型，附上实测数据（API 调用记录、费用明细、任务场景）
- 表达一个观点，说明你的样本来源和判断依据
- 每一刀砍在自己买单的靶子上，才站得住

### 二、把牌亮在桌上

前提、立场、意图先说清楚，不让人在字缝里猜。

- 开篇说明：谁是读者？这报告是写给谁看的？目的是什么？
- 如果带有立场，直接承认，不要假装客观
- 如果带有攻击性，把它放在括号里的「元拆解」摊开给人看

### 三、不要装

去掉「让我看起来很专业」的废话，只留最有信息量的部分。

- 「今天中午」比「近日，我们进行了一项研究」有力
- 「充值了 API」比「我们获取了 API 访问权限」诚实
- 「随手测的」比「在严格受控的实验条件下」可信
- 短句优先。一句话说不完的，拆成两句

### 四、画面感优先

找一个能让人「看到」的表达，而不是只丢数字。

| 不要 | 要 |
|------|-----|
| 贵了 30 倍 | 一个任务就够 DeepSeek 烧一个下午了 |
| 性价比不高 | 没有动力上生产 |
| 缺乏生态位 | 周围没人用它 |

## 报告结构（参考）

```
# 标题（结论前置，如「量潮大模型选型报告」）

## 前言
- 背景：为什么写这个报告
- 方法：怎么测的、花了多少钱
- 立场：什么态度
- （可选）对特定读者的喊话

## 正文
- 事实 > 数据 > 观点
- 先认可，再批评（能力在前，结论在后）
- 每段只做一个判断

## 结论
- 一句话说死，不给「仅供参考」的退路
- 不怕得罪人，怕的是说了等于没说
